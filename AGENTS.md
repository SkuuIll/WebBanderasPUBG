# FlagForge Studio - Agent Notes

## Purpose
FlagForge Studio is a static, client-only PWA for creating numbered PUBG Observer flag/logo/symbol packs. It runs from `index.html`, uses vanilla JavaScript, and exports ZIP files in the browser.

## Commands
- `npm test` - run the static integrity tests.
- For manual browser checks, serve the repo locally, for example: `python -m http.server 8765 --bind 127.0.0.1`.

## File Map
- `index.html` - page structure, CDN dependencies, modals, controls, export UI.
- `css/style.css` - design tokens, layout, responsive behavior, component styles.
- `js/app.js` - state, rendering, i18n, search, canvas preview, share links, save/load, export.
- `js/db.js` - country/flag database.
- `js/platforms_db.js` - platform/logo database.
- `js/symbols_db.js` - competitive symbol database for feed markers.
- `tests/run-tests.mjs` - static tests for i18n, DOM IDs, assets, manifest, CSP, and helper contracts.
- `instalar.bat` - visible template for the installer generated into exported ZIP files.
- `docs/AI_CONTEXT.md` - compressed context for future AI sessions.
- `docs/ARCHITECTURE.md` - architecture and data flow.
- `docs/UX_AUDIT.md` - UX findings and priorities.
- `docs/COMPETITIVE_MODE.md` - deterministic 100-flag competitive order and rationale.

## Editing Rules
- Keep the app framework-free: no build step, backend, or paid API.
- Do not edit `.codex/environments/environment.toml`; it is autogenerated.
- Keep Spanish as the primary UI language and keep English i18n keys in sync.
- When adding DOM IDs referenced from JS, update tests if needed.
- Avoid broad rewrites of `app.js` or `style.css` unless the task explicitly calls for refactoring.

## Verification
Always run `npm test` after changes. For UI work, also do a local browser smoke test: load the app, add an item, preview it, save/share/export, and check the console.
